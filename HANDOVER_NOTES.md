# 🛋️ RoutinHub: Webinar Cover Engine & Pipeline (Handover)

Этот проект был модернизирован: мы перешли от сложной модификации PSD к сверхбыстрому и точному движку на базе **Canvas**.

## 🎨 1. Движок Antigravity Render v2.2
**Основной файл:** `src/app/api/webinars/process/route.ts`
**Принцип работы:** 
- Вместо парсинга PSD при каждом запросе используется готовая подложка `public/templates/background.png`.
- Текст (даты) накладывается через `ctx.setTransform()` с использованием полной матрицы трансформации из Photoshop:
  `a=1.692, b=-0.176, c=0.016, d=1.700, tx=640, ty=459.4`
- Это обеспечивает **искажение (Skew)**, масштаб и наклон в точности как в оригинальном макете.
- Внедрена логика **Smart Font Resize**: шрифт автоматически уменьшается с 84px до 40px, если дата не влезает в маску.

## 🔡 2. Работа со шрифтами (Windows/Node-Canvas)
**Критически важно:** На Windows библиотека `node-canvas` очень капризна.
- **Формат**: Перешли на `.ttf` (`druktextwidecyr-medium.ttf`).
- **Схема регистрации**: Шрифт регистрируется глобально до создания любого холста под именем `AntigravityFont`.
- **Имя в коде**: `ctx.font = "84px 'AntigravityFont'"`.
- **Рекомендация**: Если шрифт «слетит», установи его в саму Windows («для всех пользователей») и перезапусти VS Code.

## ⚙️ 3. Пайплайн обработки вебинаров
**Маршруты:**
- `POST /api/webinars/upload`: Прием видео.
- `GET /api/webinars/jobs`: Список задач и статусы.
- `POST /api/webinars/process`: Ядро. Здесь происходит:
  1. Генерация обложки (Canvas).
  2. Рендеринг видео (ffmpeg).
  3. Извлечение аудио.
  4. Транскрибация (Whisper).
  5. Анализ (OpenAI Assistant).

## 🛠 4. Полезные скрипты (папка /scripts)
- `make-background.js`: Генерирует "чистый" фон из PSD (скрывая слой `text_title`).
- `test-render.js`: Ультимативный тест рендеринга. Генерирует `public/test-render-110426.png` для проверки наклона и шрифта.
- `font-debug.js`: Проверка, видит ли система установленный шрифт.

## 🛤 5. Frontend
- `/webinars`: Дашборд со списком задач.
- Реализован «тихий» поллинг (polling) с защитой от утечек памяти (`isMounted`), чтобы не перегружать сервер логами.
